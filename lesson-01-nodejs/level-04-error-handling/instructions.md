# Error Handling

## Objective

Practice handling errors with `try/catch`

## Benefits

Handing errors prevents your programs from crashing.

## Complete these tasks

1. Write the code from scratch. Do not copy/paste.
2. In `script.js`, store a value with `localStorage`.
3. Use `document.querySelector` to select a `p` tag.
4. Wrap the code with `try/catch`.
5. In the `catch` block, use console.log to display a message.
6. Let the message explain why there is an error.
7. Run the script.

## More Information

- The `try` code block attempts to run code.
- The `catch` code block runs when there is an error.
- [try/catch reference](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/try...catch)

## Usage Tips

- The `catch` statement accepts an error object. Example: `catch(error)`

## Hints

- Error objects sometimes have a `message` that can be displayed.
